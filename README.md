* **Python:** >= 3.10.
* **Виртуальное окружение:** venv.
* **IDE:** VS Code / PyCharm.
* **Особенности среды (Shared Folders)**

#### * Проект разрабатывается на Windows-хосте с монтированием папки через /media/sf_linux_share. Для корректной работы Python/Django внутри Linux-среды используется изолированное виртуальное окружение (venv), что исключает ошибки символических ссылок файловой системы NTFS.

## 🛠 Инструкция по запуску (Development)
### Шаг 1. Подготовка зависимостей
```text 
cd /media/sf_linux_share/diploma_project
source env/bin/activate
pip install -r requirements.txt
```
### Шаг 2. Инициализация БД
```bash
python manage.py makemigrations
python manage.py migrate
python manage.py createsuperuser
```

### Шаг 3. Запуск сервера
```bash
python manage.py runserver 0.0.0.0:8000
# Приложение доступно по адресу: http://127.0.0.1:8000/admin/
```

## ✅ Этап 2: «Проработка моделей данных»
### Модели данных и API
#### Реализована архитектура базы данных, REST API с документацией и разграничением прав доступа.

### 👤 Пользователи
* **Авторизация:** по Email (поле username отключено).
* **Роли:** разделение на «Покупателя» и «Магазин» (type) для ограничения импорта прайсов.
* **Профиль:** добавлены поля компании и должности.
### 📦 Товарный каталог
* **Структура:** реализована связь «Один ко многим» между базовым товаром (Product) и его конкретным предложением от магазина (ProductInfo).
* **Характеристики:** Для хранения технических характеристик (Цвет, Память, Диагональ) используется гибкое поле JSONField characteristics. Это позволяет хранить произвольные пары "ключ-значение" без создания дополнительных таблиц, что ускоряет разработку и запросы к БД.
  - Пример данных: {"Цвет": "Золотистый", "Память": "256 Гб"}.
* **Связи:** Категории привязаны к магазинам через Many-to-Many.
### 🛒 Заказы
* **Жизненный цикл:** статусы от корзины до доставки.
* **Контакты:** отдельная модель адресов и телефонов покупателя.
* **Логика:** сумма заказа рассчитывается динамически в Python. Цены закупки скрыты от публичного API.
### ⚙️ Backend (DRF)
* **Оптимизация:** настроен select_related/prefetch_related. Ошибки N+1 запросов устранены.
* **Права:** публичный просмотр каталога; доступ к заказам/контактам только после авторизации.
* **Админка:** все модели доступны в Django Admin с удобными инлайнами.
### 📄 Документация
* **Интерактивный Swagger UI доступен по адресу /docs/. Генерация схемы OpenAPI автоматическая.**
